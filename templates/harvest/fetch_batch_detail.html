What you never see on this page

  • Skipped “fresh” — labels that already had a successful fetch within HarvestEngineConfig.min_hours_since_fetch are not queued, so they are not rows here. See Harvest batch activity or grep logs for [HARVEST_AUDIT queue] (skipped_fresh).
  • Excluded labels — dead career portals (portal_alive=False), disabled platforms, or empty tenant_id are filtered out before any batch row exists.
  • After the batch finishes — the system may auto-queue URL validation and pool sync (see HarvestEngineConfig). That work is separate Celery tasks, not listed as company rows.
  • “Failed” batch counter includes SKIPPED company runs (e.g. missing harvester), not only HTTP failures.
